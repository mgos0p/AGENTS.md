## JavaScript / TypeScript ガイドライン

### コーディングスタイル

- 関数ではなくアロー関数を使用する。
- import は相対パスではなくパスエイリアス（@/〜）を優先する。
- 変数名・関数名はキャメルケースを使用する。
- 非同期処理は async/await を使用し、Promise チェーンは避ける。
- any 型の使用は避け、可能な限り型を明示する。
- TypeScript では関数の返り値型を明示する。
- 本番コードに不要な console.log を残さない。
- null / undefined は Optional Chaining（?.）や Null 合体演算子（??）を適切に使用する。
- マジックナンバー／文字列は定数として切り出す。
- デフォルトエクスポートより名前付きエクスポートを優先する。
- 未使用の変数や import を残さない。

### import 順序

1. 標準ライブラリ
2. 外部ライブラリ
3. アプリケーション共通コード
4. 相対パス

### コメント方針

- 「何をしているか」ではなく「なぜそう実装するか」を中心に記述する。

## React ガイドライン

### 基本方針

- 全てアロー関数コンポーネントを使用する。
- Props の型は type または interface でコンポーネント外に定義する。
- コンポーネント名はパスカルケースを使用する。
- やむを得ない場合を除き、useEffect / useMemo / useCallback の使用を避ける。
  - 使用する場合は依存配列を正確に記述する。

### 設計・責務分離

- UI に関係しない処理はカスタムフックとして分離する。
- 1 ファイル 1 コンポーネント（または 1 機能）を基本とする。
- 複雑なビジネスロジックはコンポーネントから切り出し、ユーティリティ関数またはカスタムフックにまとめる。
